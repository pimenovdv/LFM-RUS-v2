# Запуск пайплайна

В этом репозитории используется `click` для предоставления CLI для пайплайна обучения. Все команды запускаются через `main.py`. Мы рекомендуем использовать `uv` для управления зависимостями.

## Установка

Для установки версии с поддержкой GPU (по умолчанию):

```bash
uv sync
```

Для установки версии только для CPU:

```bash
uv sync --extra cpu
```

## Запуск этапов

Пайплайн разделен на несколько этапов, каждый из которых использует конфигурационный файл из директории `configs/`.

### 1. Обучение токенизатора и FOCUS
Обучает новый токенизатор на репрезентативной выборке данных и выполняет лексическую инициализацию словаря модели с использованием метода FOCUS.

```bash
uv run python main.py tokenizer --config configs/tokenizer.yaml
```

*Для быстрого тестирования используйте флаг `--dummy-data`:*
```bash
uv run python main.py tokenizer --config configs/tokenizer.yaml --dummy-data
```

### 2. Прунинг
Удаляет неиспользуемые токены из словаря модели на основе калибровочного датасета для оптимизации размера и эффективности.

```bash
uv run python main.py prune --config configs/pruning.yaml
```

### 3. Подготовка данных
Запускает фильтрацию (от спама, SEO и т.д.) и дедупликацию MinHash LSH на датасете предварительного обучения.

```bash
uv run python main.py data-prep --config configs/data_prep.yaml
```

### 4. Непрерывное предварительное обучение (CPT)
Продолжает фазу предварительного обучения модели с использованием подготовленного датасета.

```bash
uv run python main.py cpt --config configs/cpt.yaml
```

### 5. Контролируемое дообучение (General SFT)
Выполняет настройку модели на инструкциях с использованием диалоговых датасетов, отформатированных в ChatML.

```bash
uv run python main.py sft --config configs/sft.yaml
```

### 6. Выравнивание (DPO/GRPO)
Выравнивает модель с человеческими предпочтениями, используя такие методы, как DPO или GRPO (RLVR), с пользовательскими функциями вознаграждения.

```bash
uv run python main.py alignment --config configs/alignment.yaml
```

### 7. Финальное целевое дообучение (Task SFT)
Выполняет дообучение под конкретную задачу, при необходимости используя PEFT/LoRA.

```bash
uv run python main.py task-sft --config configs/task_sft.yaml
```

## Запуск тестов

Чтобы запустить полный набор тестов и проверить покрытие (требуются `pytest` и `pytest-cov`):

```bash
uv run pytest --cov=src --cov-fail-under=90 --cov-report=term-missing
```
