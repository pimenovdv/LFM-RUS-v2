Этот файл содержит список известных заглушек (dummy/temp), хаков (hack/fixme) и временных решений, которые были найдены в коде пайплайна обучения и требуют доработки или рефакторинга.

## Найденные проблемы

### [x] 1. Hardcoded Fallback для словаря в Tokenizer (main.py)
* **Место:** `main.py`, строка 98
* **Проблема:** Если при извлечении новых токенов не найдено строго новых токенов (отсутствующих в базе), используется хардкодный список-заглушка `["Ġкошка", "Ġсобака"]`. Это может скрыть реальные проблемы с данными во время production-обучения.
* **Что сделать:**
  - Вместо использования заглушки, скрипт должен либо падать с ошибкой `ValueError` (в production), либо корректно обрабатывать ситуацию, когда новые токены не нужны.
  - Заглушки должны использоваться строго при передаче аргумента `--dummy-data`.

### [x] 2. Временная директория для прунинга (src/pruning.py)
* **Место:** `src/pruning.py`, функция `prune_tokenizer_and_model` (строка ~35)
* **Проблема:** Для прямого редактирования JSON-структуры токенизатора используется временная папка на диске `./temp_tokenizer_pruning`, которая может не удаляться после завершения работы.
* **Что сделать:**
  - Использовать модуль `tempfile` (например, `tempfile.TemporaryDirectory()`) для гарантированной очистки временных файлов после завершения работы функции, или выполнять модификацию структуры словаря полностью в оперативной памяти (если это поддерживается библиотекой `tokenizers`).

### [x] 3. Хардкод функции вознаграждения в Rejection Sampling (src/alignment/rejection_sampling.py)
* **Место:** `src/alignment/rejection_sampling.py`, функция `evaluate_responses` (строка ~35)
* **Проблема:** При оценке ответов в режиме "Best-of-N", если имя функции вознаграждения не равно "accuracy", используется бессмысленная заглушка, зависящая от длины текста: `len(text) * 0.01` (`dummy default reward`). Модель будет просто учиться генерировать максимально длинные ответы (reward hacking).
* **Что сделать:**
  - Вынести логику Reward-функций в отдельный модуль.
  - Добавить реальные эвристики (например, проверку формата, качества кода) или использование предобученной Reward Model, если "accuracy" не подходит.

### [x] 4. Фоллбек на базовую модель в RLHF (PPO) (src/alignment/pipeline.py)
* **Место:** `src/alignment/pipeline.py`, блок `elif method == "ppo"` (строка ~380-400)
* **Проблема:** Если в конфигурации не указан `reward_model_path`, пайплайн использует базовую языковую модель (`model_name`) в качестве модели вознаграждения, что приведет к случайной инициализации классификационной головы (`randomly initialized head`) и бессмысленному обучению.
* **Что сделать:**
  - Добавить строгую проверку наличия `reward_model_path` для PPO.
  - Выбрасывать исключение, если путь не указан (исключение возможно только в режиме `dummy_data`).
