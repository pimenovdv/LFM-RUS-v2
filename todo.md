# План внедрения обучения Diffusion Language Models (MDLM) в пайплайн LFM-RUS-v2

Этот документ содержит пошаговый план разработки для интеграции маскированной дискретной диффузии в существующий пайплайн обучения. Каждый чекбокс соответствует объему работ примерно на один Pull Request (PR).

## [x] Шаг 1: Кастомный механизм внимания и конфигурация (Custom Attention & Config)
**Цель:** Создать базовые компоненты для двунаправленного (bidirectional) внимания и настроить конфигурацию модели.
* **Детали реализации:**
  * Реализовать модифицированный слой внимания (например, `ModifiedLlamaFlashAttention2`), наследующийся от оригинального. Принудительно установить `self.is_causal = False` для отключения каузальной маски.
  * Реализовать корректную обработку pad-токенов: явно перемножать матрицы внимания с инвертированной `attention_mask` (установка весов pad-токенов в $-\infty$ до softmax), чтобы избежать бага оригинальной LLaDA.
  * Создать `DiffusionConfig` (наследуется от `PretrainedConfig`). Добавить параметры: `mask_token_id`, `diffusion_steps`, `remasking_strategy`, `block_size`. Указать `model_type="lfm_masked_diffusion"` и игнорируемые ключи (например, `past_key_values`).
  * Добавить интеграцию с механизмом Hugging Face `AutoClasses`: `DiffusionConfig.register_for_auto_class()`.

## [x] Шаг 2: Класс модели и алгоритм генерации (Diffusion Model & Generation)
**Цель:** Реализовать основной класс диффузионной модели и кастомный метод итеративного демаскирования для инференса.
* **Детали реализации:**
  * Создать `DiffusionModelForConditionalGeneration` (наследуется от `PreTrainedModel`). Включает в себя Embeddings, слои трансформера с новым вниманием и LM Head.
  * Переопределить `forward()`: расчет функции потерь должен производиться исключительно по маскированным позициям токенов.
  * Переопределить метод `generate()` для итеративного сэмплирования:
    * Инициализация тензора `<MASK>` токенами.
    * Цикл от $t=1$ до $t=0$ с шагом $\Delta t = 1/N$.
    * Механизм выбора демаскирования с максимальной уверенностью (confidence-based unmasking).
    * Алгоритм низкоуверенного перемаскирования (low-confidence remasking) для исправления ошибок.
  * Оптимизация инференса: внедрение Ascending Step-Size (ASS) и EOS Early Rejection (EOSER). Реализовать block-wise сэмплирование с повторным использованием KV-кэша.
  * Регистрация модели `DiffusionModelForConditionalGeneration.register_for_auto_class("AutoModelForCausalLM")` и регистрация в локальной сессии (`AutoConfig.register`, `AutoModelForCausalLM.register`).

## [x] Шаг 3: CPT пайплайн и WSD-обучение с Block Diffusion
**Цель:** Адаптировать этап Continual Pre-Training для перевода AR-модели в базовую диффузионную (Base Diffusion).
* **Детали реализации:**
  * Внедрить поддержку расписания Warmup-Stable-Decay (WSD) со сверхнизким learning rate на этапе warmup для смягчения дрейфа весов.
  * Добавить механизм Top-k Checkpoint Merge для усреднения весов последних итераций CPT.
  * Реализовать архитектуру Block Diffusion (BD3LM) во время обучения: разбиение последовательности на блоки (например, по 64 токена). Внутри блока действует двунаправленное диффузионное внимание, между блоками — причинно-следственная связь (causal) для сохранения возможности KV-caching.

## [x] Шаг 4: SFT пайплайн и SFT-маскирование
**Цель:** Модифицировать пайплайн Supervised Fine-Tuning для создания Instruct-модели.
* **Детали реализации:**
  * Реализовать логику SFT-маскирования: токены промпта всегда остаются открытыми ($t=0$), токены ответа маскируются с вероятностью $t \sim \mathcal{U}$.
  * Адаптировать `forward()` для SFT: функция потерь $\mathcal{L}_{SFT}$ рассчитывается только на замаскированных позициях ответа (игнорируя промпт и открытые токены ответа).
  * Внедрить Blockwise SFT для консистентности с этапом CPT, сохраняя локальный двунаправленный контекст и глобальный AR-кэш инструкций.

## [ ] Шаг 5: Alignment с использованием CJ-GRPO / VRPO
**Цель:** Заменить стандартные AR-алгоритмы RLHF на алгоритмы, совместимые с диффузионными траекториями.
* **Детали реализации:**
  * Интегрировать алгоритм CJ-GRPO (Consistency Trajectory Group Relative Policy Optimization) или Variance-Reduced Policy Optimization (VRPO) в пайплайн выравнивания (Alignment).
  * Реализовать согласование между траекторией rollout (параллельного демаскирования) и траекторией оптимизации политики для устранения skip-step ошибок.
  * Адаптировать вознаграждения (Reward Modeling) для оценки параллельно генерируемых диффузионных траекторий.

## [ ] Шаг 6: Доменная специализация (Controllability & Steering)
**Цель:** Внедрить механизмы управления генерацией для специализированных доменных моделей.
* **Детали реализации:**
  * Реализовать дискретный Classifier-Free Guidance (CFG) с нормализацией столбцов матрицы интенсивностей (rate matrix column normalization) для устранения фрагментации текста и обеспечения строгих доменных ограничений.
  * Реализовать механизмы активационного управления (Activation Steering): извлечение векторов направлений (directions) из контрастивных промптов (forward pass).
  * Добавить применение вектора управления (глобальная интервенция) к активациям residual-слоев во время обратной диффузии.
  * Убедиться в корректной поддержке кросс-языкового трансфера (перенос векторов ограничений с английских данных на русскоязычную генерацию).
