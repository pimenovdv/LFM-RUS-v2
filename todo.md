# План реализации методов выравнивания (Alignment)

На основе документа `docs/extra_alignment_methods.md`, необходимо расширить функционал пайплайна (в `src/alignment/pipeline.py` и конфигурациях), добавив поддержку следующих методов обучения.

## 1. Методы прямой оптимизации (Direct Optimization)
Методы, не требующие отдельной Reward Model, решающие проблемы нестабильности PPO и снижающие потребление памяти.

- [x] **Реализовать IPO (Identity Preference Optimization)**
  - Добавить поддержку `loss_type="ipo"` в текущий класс `DPOTrainer` / `DPOConfig` (библиотека `trl` поддерживает это из коробки).
  - Добавить обработку параметра `method: ipo` в `run_alignment_pipeline`.
- [ ] **Реализовать KTO (Kahneman-Tversky Optimization)**
  - Настроить формат датасета для одиночных ответов с бинарной меткой (👍 / 👎).
  - Интегрировать класс `KTOTrainer` из библиотеки `trl`.
  - Добавить обработку параметра `method: kto` в `run_alignment_pipeline`.
- [ ] **Реализовать ORPO (Odds Ratio Preference Optimization)**
  - Использовать датасет с парами предпочтений (как в DPO).
  - Интегрировать класс `ORPOTrainer` из библиотеки `trl`.
  - Добавить обработку параметра `method: orpo` в `run_alignment_pipeline`.
- [ ] **Реализовать CPO (Contrastive Preference Optimization)**
  - Подготовить конфигурацию для использования `CPOTrainer` из `trl` (полезно для генерации кода).
  - Добавить обработку параметра `method: cpo`.

## 2. Дата-центричные и Self-Play методы
Подходы, использующие генерацию данных самой моделью.

- [ ] **Реализовать Rejection Sampling (Best-of-N / RFT)**
  - Разработать скрипт инференса для генерации N (например, 64) вариантов ответа на промпты.
  - Настроить пайплайн оценки вариантов с помощью Reward Model (или эвристик).
  - Формировать SFT-датасет из лучших ответов и дообучать модель, используя существующий пайплайн SFT (`src/sft.py` или `task_sft.py`).
- [ ] **Реализовать SPIN (Self-Play Fine-Tuning)**
  - Подготовить пайплайн для генерации ответов самой моделью на основе SFT-промптов.
  - Интегрировать класс `SPINTrainer` (или его аналог/DPO-модификацию) из библиотеки `trl`.
  - Добавить параметр `method: spin` в `run_alignment_pipeline`.

## 3. Методы на основе обучения с подкреплением (RL)
Классические методы RL, обеспечивающие высокий уровень Exploration, но требующие больше ресурсов.

- [ ] **Реализовать RLHF (PPO)**
  - Написать скрипт для обучения отдельной Reward Model (`RewardTrainer` из `trl`).
  - Интегрировать `PPOTrainer` из `trl` для выравнивания с использованием Reward Model.
  - Настроить управление памятью для 4-х моделей (Policy, Reference, Reward, Value).
- [ ] **Реализовать RLAIF (Constitutional AI)**
  - Создать пайплайн генерации датасета предпочтений с помощью мощной LLM-судьи на основе заданных "конституционных" правил.
  - Интегрировать генератор в существующий процесс подготовки данных или напрямую перед RLHF/DPO.
