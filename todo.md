# Пайплайн обучения языковой модели (LFM)

## 1. [x] Обучение токенизатора и лексическая инициализация (FOCUS)
*   **Задача**: Обучение нового токенизатора (BPE/Unigram) на целевом датасете (40% ru, 30% en, 15% code, 15% math) с посимвольным разбиением цифр. Инициализация новых токенов с использованием алгоритма FOCUS (fastText + sparsemax) для стабильного "теплого старта".
*   **Конфигурация**: `configs/tokenizer.yaml`

## 2. Прунинг и оптимизация токенизатора
*   **Задача**: Удаление избыточных токенов из словаря базовой модели (например, Llama 3) на основе калибровочного корпуса целевых языков. Перегенерация списка слияний (merges) и изменение размера матрицы эмбеддингов в модели.
*   **Конфигурация**: `configs/pruning.yaml`

## 3. Очистка и дедупликация данных (MinHash LSH)
*   **Задача**: Подготовка датасета для Continual Pre-Training. Запуск конвейера дедупликации на базе MinHash и LSH (используя DataTrove) с возможной GPU-акселерацией (cuDF/Dask) для предотвращения doom looping.
*   **Конфигурация**: `configs/data_prep.yaml`

## 4. Continual Pre-Training (CPT)
*   **Задача**: Продолженное предобучение модели на собранном корпусе (~101B токенов) с учетом различных доменов (ru, en, code, math). Настройка гиперпараметров обучения и чекпоинтирования.
*   **Конфигурация**: `configs/cpt.yaml`

## 5. Supervised Fine-Tuning (General SFT)
*   **Задача**: Настройка инструктивного поведения модели. Обучение в формате диалогов (ChatML) с использованием `SFTTrainer` из библиотеки `trl` и применением функции упаковки последовательностей (sequence packing) для оптимизации вычислений.
*   **Конфигурация**: `configs/sft.yaml`

## 6. Выравнивание (Alignment): DPO / RLVR (GRPO)
*   **Задача**: Тонкая настройка предпочтений модели. Использование DPO для общих диалоговых задач и GRPO (RLVR) с детерминированными функциями награды (accuracy_reward, variance_reward) для задач программирования и математики.
*   **Конфигурация**: `configs/alignment.yaml`

## 7. Финальное дообучение (Task SFT)
*   **Задача**: Специализация модели под узкие задачи. Выполнение Full Fine-Tuning (для моделей < 350M) или LoRA-адаптации (для более крупных) с таргетированием специфических слоев модели (включая мультипликативные гейты для архитектуры LFM).
*   **Конфигурация**: `configs/task_sft.yaml`
