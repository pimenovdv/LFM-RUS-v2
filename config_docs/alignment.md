# Настройки Alignment (alignment.yaml)

Этот файл содержит конфигурацию для этапа выравнивания модели (alignment). В пайплайне поддерживаются различные методы для обучения с подкреплением и оптимизации предпочтений.

## Параметры

* **`stage`** (строка): Определяет этап пайплайна.
  * *Пример*: `"alignment"`
* **`method`** (строка): Метод выравнивания, который будет использоваться. Доступные варианты:
  * `"dpo"` (Direct Preference Optimization): Оптимизирует модель напрямую на наборе данных предпочтений (нужны пары `chosen` и `rejected`). Не использует явную модель вознаграждения.
  * `"kto"` (Kahneman-Tversky Optimization): Оптимизирует модель на основе бинарной оценки одиночных ответов (нравится/не нравится), не требуя пар предпочтений.
  * `"grpo"` (Generalized Reward Policy Optimization): Использует функции вознаграждения (reward functions) для оценки сгенерированных ответов и оптимизации политики генерации. Ожидает только `prompt` в наборе данных.
* **`learning_rate`** (число с плавающей точкой): Скорость обучения.
  * *Пример*: `1.0e-5`
* **`batch_size`** (целое число): Размер пакета (batch size) для обучения.
  * *Пример*: `4`
* **`max_prompt_length`** (целое число): Максимальная длина промпта в токенах.
  * *Пример*: `512`
* **`max_completion_length`** (целое число): Максимальная длина генерируемого ответа в токенах.
  * *Пример*: `1024`
* **`epochs`** (целое число): Количество эпох обучения.
  * *Пример*: `1`
* **`num_generations`** (целое число): Количество генераций на каждый промпт (используется только в GRPO для оценки распределения).
  * *Пример*: `4`
* **`reward_funcs`** (список строк): Список функций вознаграждения, используемых для оценки качества генерации (используется только в GRPO). Доступные варианты:
  * `"accuracy"`: Проверяет наличие правильного ответа (например, тега `<solution>`) в тексте. Выдает положительное вознаграждение `1.0` при наличии и отрицательное `-1.0` при отсутствии.
  * `"variance"`: Штрафует модель за зацикливания или ответы с низкой вариативностью. Штраф `-2.0` за повторение n-грамм, вознаграждение `0.5` за разнообразный текст.
* **`dataset_path`** (строка): Путь к датасету или формат данных.
  * *Пример*: `"json"`

## Пример использования
```yaml
stage: alignment
method: "grpo"
learning_rate: 1.0e-5
batch_size: 4
max_prompt_length: 512
max_completion_length: 1024
epochs: 1
num_generations: 4
reward_funcs:
  - "accuracy"
  - "variance"
dataset_path: "json"
```
