# Финальное целевое дообучение (Task SFT)

## Формат данных
На этапе Task SFT модель дообучается под узкую, конкретную задачу (например, экстракция сущностей, SQL-генерация, классификация текстов с жестким JSON-выходом).
Формат данных аналогичен этапу General SFT — диалоговый (ChatML), но промпты и ответы строго соответствуют целевой задаче.
Если используется PEFT/LoRA (что типично для Task SFT, чтобы не разрушить обобщающие способности базовой модели), это указывается в конфигурации `task_sft.yaml`.

Пример JSONL для Task SFT (экстракция):
```json
{"messages": [{"role": "system", "content": "Извлеки имена из текста в формате JSON."}, {"role": "user", "content": "Иван пошел в магазин с Марией."}, {"role": "assistant", "content": "{\"names\": [\"Иван\", \"Мария\"]}"}]}
```

## Требуемое количество данных
Количество данных для Task SFT сильно зависит от сложности узкой задачи, но обычно оно значительно меньше, чем для General SFT. Основная задача — заставить модель принять определенный формат вывода.

Оценки для узкого Task SFT (с использованием полного fine-tuning или LoRA):

* **100m**: **~500 - 2,000** примеров. (Мелкие модели требуют больше данных для фиксации узкого формата, так как у них слабее in-context learning).
* **350m**: **~1,000 - 3,000** примеров.
* **0.5b**: **~1,000 - 4,000** примеров.
* **1b**: **~2,000 - 5,000** примеров.
* **2b**: **~2,000 - 8,000** примеров.
* **4b**: **~3,000 - 10,000** примеров.
* **8b**: **~5,000 - 15,000** примеров.

*(Если целевая задача крайне сложна — например, написание сложных SQL запросов к базе из 100 таблиц — требуемый объем данных может возрасти до 50k-100k примеров для крупных моделей, но для большинства бизнес-задач достаточно 1k-10k качественных примеров).*
